忘记Ubuntu密码如何重新设置
进入系统时选择：Advanced options for Ubuntu
然后选择带( recovery mode)的选项.
进入恢复模式后选择：root Drop to root shell prompt.
然后分为以下两种情况：

a. 若直接输入：passwd (则为修改root密码, 无论你之前是否设置了root密码，因为没有提示需要输入之前的root密码，所以可以直接修改)
输入新密码：#
再输入一遍密码：#

b. 若输入： passwd yujiao （ 则为修改一般用户密码，yujiao是系统创建过的用户）
输入新密码：#
再输入一遍密码：#

看到password updated successfully即为成功
最后记得输入reboot重启，如果直接输入exit有可能没有保存刚才的设置。

解决关于安装xLearn在运行时报错std::logic_error的解决办法:
solution refers to：
https://github.com/aksnzhy/xlearn/issues/115
https://www.cnblogs.com/peida/archive/2013/01/08/2850483.html
首先要创建一个crontab文件，然后在文件里进行配置，示例如下：

打开文件：
vim /etc/crontab
输入命令：
SHELL=/bin/bash
PATH=/usr/local/bin/:/usr/local/bin/:/usr/local/bin:/usr/local/bin:/root/miniconda3/bin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin
MAILTO=""HOME=/
export USER=yujiao

crontab里是系统命令，side-packages里主要是Python的包和库
/ 是最高级的目录
运行其它二进制的程序也一样，直接写vim /etc/crontab，linux系统会去PATH里寻找有没有叫crontab的，除此之外/bin, /sbin, /usr/bin，/usr/sbin等也在PATH里.如果在当前目录运行一个test.sh文件，就要用./test.sh告诉系统说，就在当前目录找。

xLearn所需数据格式
对于 LR 和 FM 算法而言，我们的输入数据格式必须是 CSV 或者 libsvm. 对于 FFM 算法，我们的输入数据必须是 libffm 格式:

libsvm format:

   y index_1:value_1 index_2:value_2 ... index_n:value_n

   0   0:0.1   1:0.5   3:0.2   ...
   0   0:0.2   2:0.3   5:0.1   ...
   1   0:0.2   2:0.3   5:0.1   ...

CSV format:

   y value_1 value_2 .. value_n

   0      0.1     0.2     0.2   ...
   1      0.2     0.3     0.1   ...
   0      0.1     0.2     0.4   ...

libffm format:

   y field_1:index_1:value_1 field_2:index_2:value_2   ...

   0   0:0:0.1   1:1:0.5   2:3:0.2   ...
   0   0:0:0.2   1:2:0.3   2:5:0.1   ...
   1   0:0:0.2   1:2:0.3   2:5:0.1   ...

xLearn 还可以使用 , 作为数据的分隔符:

libsvm format:

   label,index_1:value_1,index_2:value_2 ... index_n:value_n

CSV format:

   label,value_1,value_2 .. value_n

libffm format:

   label,field_1:index_1:value_1,field_2:index_2:value_2 ...

Linux SSH 远程登录错误解决办法 WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!
参考资料：https://blog.csdn.net/nahancy/article/details/51052127
将known_hosts文件中的与登录错误的IP的公钥删除即可,known_hosts是记录远程主机的公钥

进入文件：
vim .ssh/known_hosts
删除相关地址host即可
